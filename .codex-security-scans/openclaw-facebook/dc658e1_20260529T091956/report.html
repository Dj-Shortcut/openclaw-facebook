Findings

High: WhatsApp Webhook Deliveries Bypass Replay Protection

apps/image-gen/server/_core/whatsappWebhook.ts:134 processes normalized WhatsApp events without the replay claim used by Facebook events. A previously valid signed body/header can be replayed to repeat state changes, replies, media persistence, or generation work.

Medium: Single-Target Facebook Webhook Fallback Accepts Unmatched Page Events

src/monitor.ts:713 falls back to the sole target even when recipient.id is present and mismatched, allowing wrong-Page signed events to be processed under the configured account.

Medium: Concurrent WhatsApp Generation Requests Can Bypass Daily Quota

apps/image-gen/server/_core/whatsappFlows/styleGenerationFlow.ts:276 checks quota before generation and increments after success without the per-sender guard used by Messenger generation.

Medium: Data Deletion Loses Retry State After Storage Delete Failure

apps/image-gen/server/_core/dataDeletionService.ts:31 swallows storage delete failures and :71 clears state, potentially orphaning private media after a deletion request.

Medium: WhatsApp Media Download Buffers Full Response Without Local Limits

apps/image-gen/server/_core/whatsappApi.ts:221 calls arrayBuffer() without a local size cap or timeout before storing the media.
